# Crawl4AI Market Search Adapter Module

## Purpose

Crawl4AI Market Search Adapter turns a Boss Idea market query pack into
public-source crawl evidence that can feed the existing market research
collector.

The adapter exists because executive brainstorming often needs rapid competitor
and solution discovery, not only manually curated search results. It still
preserves the Boss Idea authority model: crawled output is evidence only,
research artifacts must cite sources, and implementation requires approved
manifest artifacts.

## Scope

Active scope:

- consume `agentic/runs/<run-id>/market-research-query-pack.yaml`;
- crawl only public, allowed URLs;
- normalize Crawl4AI markdown output into the existing
  `boss-idea-market-search` result contract;
- support local fixture HTML for deterministic golden tests;
- enforce source URL, domain, page count, timeout, content size, and evidence
  storage limits;
- keep raw crawl output under ignored run evidence;
- call `scripts/collect-boss-idea-research.sh` only after result normalization
  passes validation.

Deferred scope:

- paid market databases;
- social media firehose search;
- authenticated crawling;
- customer-internal or private docs;
- hosted crawler fallback such as Firecrawl;
- hosted LLM extraction;
- automatic roadmap, budget, or implementation approval.

## Workflow

```text
boss idea run
  -> collect-boss-idea-research.sh --dry-run
  -> market-research-query-pack.yaml
  -> crawl-boss-idea-market.sh
  -> Crawl4AI public page crawl
  -> ignored raw markdown evidence
  -> normalized search-results.yaml
  -> collect-boss-idea-research.sh
  -> validated market-research.md
  -> decision memo / feasibility / POC-MVP planning
```

The adapter does not choose final competitors by itself. It creates a cited,
reviewable candidate source set that Market Research Lead and Staff+ reviewers
can accept, reject, or supplement.

## Artifact Schema

Input artifact:

- `agentic/runs/<run-id>/market-research-query-pack.yaml`
- generated by `scripts/collect-boss-idea-research.sh --dry-run <run-id>`
- contains `queries[]` with `id`, `query`, and `purpose`

Normalized output artifact:

- `agentic/runs/<run-id>/market-search-results.yaml`
- conforms to `agentic/schemas/boss-idea-market-search.schema.yaml`
- each result records:
  - `id`
  - `query_id`
  - `title`
  - `source_type`
  - `access_date`
  - `reference`
  - optional `url`
  - `signal`
  - short summarized `claim`

Ignored raw evidence:

- `agentic/runs/<run-id>/crawl4ai/raw/<source-id>.md`
- `agentic/runs/<run-id>/crawl4ai/crawl-log.yaml`
- raw evidence must never be copied into tracked docs.

## CLI / Manifest / Pipeline Contract

Future command:

```bash
scripts/crawl-boss-idea-market.sh --dry-run <run-id>
scripts/crawl-boss-idea-market.sh <run-id> --seeds <seeds.yaml> --output agentic/runs/<run-id>/market-search-results.yaml
scripts/crawl-boss-idea-market.sh <run-id> --from-query-pack --output agentic/runs/<run-id>/market-search-results.yaml
```

Expected contract:

- requires an existing Boss Idea planning manifest;
- requires `boss_idea_intake`;
- reads query pack generated by `collect-boss-idea-research.sh --dry-run`;
- reads seed URLs from `--seeds` or a future search provider adapter;
- refuses URLs that are not `http` or `https`;
- refuses localhost, link-local, private RFC1918, metadata-service, and
  non-allowlisted internal targets;
- writes Crawl4AI raw markdown only under ignored run paths;
- writes normalized search results YAML;
- optionally invokes `collect-boss-idea-research.sh` as a follow-on step;
- records crawl metadata in `boss_idea_market_crawl` manifest metadata;
- does not approve artifacts, decisions, budget, or implementation.

Hermes action:

```text
id: crawl_boss_idea_market
mode: planning
authorization.action: artifact.generate
```

Pipeline registration must list the command in executable validation and must
not make live network crawl part of default golden fixtures.

## Crawl4AI Contract

Use Crawl4AI as a local-first crawler. Official Crawl4AI documentation describes
`AsyncWebCrawler` as the async crawler API and documents that it fetches pages
and can produce markdown output through crawler configuration. It also exposes
the `crwl` CLI for simple crawling and markdown output.

Implementation should prefer the Python API over shelling out to `crwl` when
we need strict per-URL policy enforcement, structured result capture, and
testable dependency injection.

Required runtime controls:

- `BrowserConfig(headless=True)`;
- `CrawlerRunConfig` with explicit timeout and cache policy;
- no LLM extraction strategy;
- no automatic provider API tokens;
- max pages per query;
- max bytes or markdown characters per page;
- stable user agent identifying this internal research tool;
- structured crawl log with success, skipped, and failed URLs.

## Failure Behavior

Block crawling when:

- run id is invalid;
- manifest is missing or not `boss-idea-response`;
- query pack is missing or malformed;
- seed URL is not `http` or `https`;
- URL resolves to private, loopback, link-local, multicast, or metadata-service
  address;
- host is not allowlisted when an allowlist is configured;
- `robots` or site policy disallows crawl for the selected path;
- page exceeds content size or timeout limits;
- Crawl4AI returns no usable markdown;
- normalized results lack competitor or mainstream-practice signals;
- claim text is multiline, too long, or appears to be copied raw source text;
- raw evidence would be written to tracked paths.

Escalate rather than block when:

- crawl returns too few sources but there is at least one cited public source;
- a relevant source requires authentication;
- source freshness is insufficient and needs human caveat;
- competitor naming is ambiguous.

## Validation Strategy

Validation must include:

- bash syntax for wrapper scripts;
- Python unit tests for URL policy and result normalization;
- fixture crawl using local HTML or `raw://` input, not the public internet;
- negative tests for SSRF targets:
  - `http://localhost`
  - `http://127.0.0.1`
  - `http://169.254.169.254`
  - private RFC1918 addresses
  - path traversal in output paths
- negative tests for excessive page count and content length;
- negative tests for missing required signals;
- validation of generated `market-search-results.yaml`;
- validation of downstream `market-research.md`;
- privacy scan over tracked files;
- AIT plus Claude Code review.

Default golden fixtures must not depend on external network availability.
Live crawl smoke tests may be separate and explicitly opt-in.

## Test Cases

Positive tests:

- local fixture page normalizes to a competitor signal;
- local fixture page normalizes to a mainstream-practice signal;
- generated search results pass `collect-boss-idea-research.sh`;
- raw markdown evidence stays under ignored run directory;
- manifest records crawl metadata without approving artifacts.

Negative tests:

- missing query pack fails;
- missing seed URLs fails;
- unsupported scheme fails;
- localhost and metadata-service URLs fail;
- non-allowlisted host fails;
- oversized page fails;
- generated result with missing reference fails;
- generated result with bad source type fails;
- generated result with missing required signal fails;
- output outside run directory fails.

## Acceptance Criteria

- Boss Idea run can produce a query pack and crawl public seed URLs through
  Crawl4AI.
- Crawl output is normalized into the existing market-search result schema.
- Normalized results can feed `collect-boss-idea-research.sh` without manual
  editing.
- Live crawl is optional and never required for deterministic golden fixtures.
- SSRF and unsafe URL classes are blocked before Crawl4AI receives the URL.
- Raw crawl evidence remains ignored and public-safe summaries remain tracked
  only when explicitly generated.
- No crawl or search output can approve artifacts, decisions, or implementation.

## Doc Review Standard

Claude Code review must verify:

- Crawl4AI role is limited to local-first public-source evidence gathering;
- live web crawling is not part of default deterministic validation;
- the design does not imply hosted LLM extraction or Firecrawl fallback;
- URL policy, output path, and raw-evidence boundaries are explicit;
- the adapter feeds the existing BIR-09 collector contract.

## Code Review Standard

Implementation review must verify:

- URL policy runs before network access;
- DNS/IP checks block private and metadata-service targets;
- timeouts, page limits, and content limits are enforced;
- Crawl4AI errors are recorded without leaking raw output to tracked files;
- generated results pass `collect-boss-idea-research.sh`;
- golden fixtures use local HTML or controlled fixture inputs;
- live crawl smoke tests are opt-in and clearly labeled.

## Rollback

Rollback must remove:

- `scripts/crawl-boss-idea-market.sh`;
- any Python helper used only by this adapter;
- Crawl4AI-specific fixtures;
- Hermes action and pipeline references;
- ignored crawl output under affected runs.

Existing BIR-09 collector and validator should remain intact because they are
provider-neutral and useful without Crawl4AI.
